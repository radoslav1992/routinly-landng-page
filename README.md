# Routinly landing page

Marketing site for [Routinly](https://routinly.org) — an AI-powered SOP generator that turns
screen recordings into step-by-step documentation. Built with [Astro](https://astro.build) as a
fully static, SEO-optimized site and deployed to Cloudflare Workers (static assets).

## Stack

- **Astro 7** — static output, zero JS by default (only the mobile nav toggle and the contact
  form ship a few lines of script)
- **Self-hosted fonts** — Bricolage Grotesque + Instrument Sans via Fontsource (no
  render-blocking Google Fonts requests)
- **@astrojs/sitemap** — `sitemap-index.xml` generated on build, referenced from `robots.txt`
- **Cloudflare Workers** — assets-only Worker (`wrangler.jsonc`), no server code

## SEO

- Per-page `<title>`, meta description, and canonical URL
- Open Graph + Twitter card tags with a branded 1200×630 share image (`public/og.png`)
- JSON-LD structured data: `Organization` + `WebSite` site-wide, `SoftwareApplication` on the
  landing page, `FAQPage` on pricing, `Blog`/`BlogPosting` on the blog
- Semantic HTML, one `h1` per page, `sitemap-index.xml`, `robots.txt`, custom 404

## Develop

```sh
npm install
npm run dev        # local dev server
npm run build      # static build into dist/
npm run preview    # serve the built site through wrangler (workers runtime)
```

## Deploy to Cloudflare

One-time: authenticate wrangler (`npx wrangler login`, or set `CLOUDFLARE_API_TOKEN` +
`CLOUDFLARE_ACCOUNT_ID` in CI).

```sh
npm run deploy     # astro build && wrangler deploy
```

The site deploys as a static-assets Worker named `routinly-landing-page`. Attach the
`routinly.org` custom domain to the Worker in the Cloudflare dashboard (Workers & Pages →
routinly-landing-page → Settings → Domains & Routes). Alternatively, connect this repo in the
Cloudflare dashboard with Workers Builds — build command `npm run build`, deploy command
`npx wrangler deploy`.

## Content

- **Blog posts** live in `src/content/blog/*.md` — frontmatter is validated by
  `src/content.config.ts`. Add a markdown file and it appears on `/blog` automatically.
- **Changelog entries** are the `entries` array in `src/pages/changelog.astro`.
- **Pricing plans / FAQ** are data arrays at the top of `src/pages/pricing.astro`.
- **Placeholder art** — the hero app preview and the dark-band SOP preview are decorative SVG
  components (`src/components/AppPreview.astro`, `SopPreview.astro`) standing in for real app
  screenshots; swap them for `<img>` captures when available. Blog covers and the OG card are
  generated by `node scripts/generate-assets.mjs`.

## Design source

The site implements the Claude Design handoff bundle
(`Routinly landing page redesign`): cream `#FAF6EE` background, ink `#1B1712`, burnt-orange
accent `#C94F1C`, Bricolage Grotesque display type, Instrument Sans body type. Design tokens
live in `src/styles/global.css`.
